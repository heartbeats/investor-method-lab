# TASK-0004｜明确边界与输入

## 对应需求
- `REQ-0002` 同一机会包展示机会验真结果与 DCF 估值联动

## 目标
- 明确这条需求当前要整合哪些现有产物、输出到哪里、哪些事情暂时不做。

## 现有输入
- `docs/top20_opportunity_pack_real_3markets.md`：当前三市场机会包正文
- `output/opportunity_validation_latest.json`：机会验真 latest 结果
- `output/opportunity_validation_positions_latest.json`：持仓/验证明细
- `data/opportunity_signal_ledger.jsonl` 与 `output/opportunity_signal_ledger_latest.json`：信号账本与最新摘要
- `docs/dcf_special_focus_daily.md` 与 DCF 字段：估值联动的重点观察口径
- `scripts/run_real_pack_3markets.sh`：当前主编排

## 目标输出
- 第一阶段先落到统一机会包口径：优先复用 `docs/top20_opportunity_pack_real_3markets.md` 或其生成链路
- 对单个标的，至少能同时看到：机会状态 / 关键验证结论 / 估值来源与关键值 / 风险提示
- 后续如需单独 JSON 或表格，可作为扩展输出，但不替代正文机会包

## In Scope
- 把机会验真结果和估值联动信息放进同一份机会包叙事
- 允许 DCF 缺失时降级到 external valuation 或 close，并保留来源说明
- 输出口径优先围绕“关键结论可解释”而不是只堆字段

## Out of Scope
- 不做交易执行或自动下单
- 不要求所有行情/估值源都实时成功；允许降级但必须显式标注
- 不在本任务里完成字段级来源映射/可信度评分的最终实现（这属于 `REQ-0003`）

## 验证
- `requirements.md` 已补充真实目标、用户、非目标、约束
- `REQ-0002` 已补充更具体的验收口径与 KPI 关联
- `TASK-0004` 已标记 done，后续 `TASK-0005` 可直接进入实现
